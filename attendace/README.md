# 差勤管理查詢系統 (Attendance Management System)

這是一個專為差勤數據分析與展示設計的前端管理系統，支援民國紀年輸入、月份篩選以及自動化的統計分析邏輯。

## 📁 檔案架構說明

| 檔案名稱 | 說明 |
| :--- | :--- |
| `attendance.html` | **前端主程式**。包含 UI 介面、數據渲染邏輯與動態篩選引擎。 |
| `attendance_setup.sql` | 資料庫結構定義檔（SQL），用於建立後端存取所需的資料表。 |
| `generate_data.py` | 原始數據生成腳本，模擬生成差勤紀錄。 |
| `inject_data.py` | 數據注入腳本，將生成的數據處理並匯入系統。 |
| `generate_summary_csv.py` | 生成統計摘要資訊的 CSV 檔案。 |
| `attendance_summary.csv` | 差勤數據摘要，供系統展示或備份使用。 |
| `attendance_system_prompt.md` | AI 開發協作指令檔，定義了系統的開發規範與邏輯約束。 |

## 🛠 核心業務規範

系統嚴格遵循以下邏輯規範進行數據處理：

### 1. 紀年與日期格式
- **民國紀年**：輸入格式為 `YYYMM`（如 11401 代表民國 114 年 1 月）。
- **轉換公式**：西元年 = 民國年 + 1911。

### 2. 數據篩選邏輯 (Filtering Rules)
- **缺勤統計 (Absence)**：
    - 範圍：僅限 **當選取月份**。
    - 展示門檻：缺勤次數 **大於等於 3 次**。
- **病假/事假統計 (Sick/Personal Leave)**：
    - 範圍：**年度累計 (Year-to-Date, YTD)**。從該年度 1 月起累計至所選月份。
    - 展示門檻：累計天數 **大於等於 10 天**。

### 3. UI 交互規範
- **反應式更新 (Reactive UI)**：切換月份時，系統必須自動重新計算統計數據並刷新當前搜尋結果，不需重新點擊查詢按鈕。
- **數據完整性**：所有統計計算均在記憶體中進行，嚴禁修改原始資料陣列。

## 🚀 快速上手
1. 開啟 `attendance.html` 即可進入查詢介面。
2. 系統會根據當前日期自動初始化月份篩選器。
3. 透過「單位代碼」或「區處搜尋」快速定位特定對象。
